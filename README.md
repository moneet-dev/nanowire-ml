# nanowire-ml

**Reproducing and extending Raya et al. 2025 — sequence-only machine-learning
prediction of microbial nanowire proteins.**

Microbial nanowires are conductive protein filaments that bacteria (e.g.
*Geobacter*, *Shewanella*, and sulfate-reducing bacteria) use for extracellular
electron transfer. Because the two structural classes — aromatic e-pili and
multiheme c-type cytochromes — share no sequence homology, BLAST fails to find
them, motivating a physicochemical-feature ML classifier. This repo reproduces
the binary nanowire-vs-not classifier of Raya et al. 2025 and extends it with
calibration, uncertainty quantification, systematic feature ablation, ESM-2
protein-language-model embeddings, and an autoresearch-style optimization loop.

> Reproduce-and-extend target:
> Raya D, Peta V, Bomgni A, et al. *Prediction and validation of nanowire
> proteins in Oleidesulfovibrio alaskensis G20 using machine learning and
> feature engineering.* Comput Struct Biotechnol J. 2025;27:1706–1718.
> [doi:10.1016/j.csbj.2025.04.022](https://doi.org/10.1016/j.csbj.2025.04.022)

## Reproduction targets (paper Table 2)

| Model | Accuracy | ROC-AUC |
|---|---|---|
| SVM | 0.9487 | 0.9696 |
| Random Forest | 0.9669 | 0.9826 |
| XGBoost | 0.9665 | 0.9857 |
| Logistic Regression | 0.9605 | 0.9903 |
| MLP | 0.9613 | 0.9920 |

Our contributions on top of reproduction: PR-AUC and ECE (calibration) for every
model, bootstrap confidence intervals, hyperparameter tuning, a stacking
ensemble, SHAP feature attribution, a feature-group ablation, an ESM-2 vs.
iFeature comparison, and an autonomous experiment loop over `src/classify.py`.

## Repository layout

```
nanowire-ml/
├── src/                  # importable modules
│   ├── data_utils.py     # FASTA loading, labeling, dataset assembly
│   ├── feature_utils.py  # iFeature (iFeatureOmegaCLI) extraction wrappers
│   ├── eval_utils.py     # ROC/PR-AUC, ECE, reliability, bootstrap CIs
│   └── classify.py       # ← the autoresearch target file
├── scripts/              # runnable entry points (one per phase)
├── data/
│   ├── raw/              # merged.fasta + labels.csv (gitignored; regenerated)
│   └── features/         # descriptor CSVs + master_features.csv (gitignored)
├── results/
│   ├── metrics/          # result CSVs (tracked)
│   └── figures/          # plots (gitignored; regenerated)
├── program.md            # autoresearch agent instructions
└── p2/                   # placeholder for the P2 KG + GNN project
```

`data/features/` and `results/figures/` are gitignored and regenerated by the
scripts — only source, result CSVs, and docs are tracked, which forces
reproducibility.

## Quickstart

```bash
pip install -r requirements.txt

# Phase 1 — build merged.fasta + labels.csv and print sanity checks
python scripts/01_data_prep.py

# Phase 2 — extract the 22 iFeature descriptors -> data/features/master_features.csv
python scripts/02_feature_extraction.py

# Phase 3 — reproduce the paper's Table 2 with stratified 5-fold CV
python scripts/03_reproduce_baseline.py

# Autoresearch target — run a single experiment and print the benchmark
python src/classify.py
```

The dataset ships with the official repo
[`bicbioeng/nanowire-protein-prediction`](https://github.com/bicbioeng/nanowire-protein-prediction)
(`Prediction dataset/`), which mirrors the Mendeley release
[doi:10.17632/vdvdrj3k2p.1](https://doi.org/10.17632/vdvdrj3k2p.1).
